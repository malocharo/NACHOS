Detailed Description

Defines an ACIA (Asynchronous Communication Interface Adapter) device driver.

The class DriverACIA defines the handler of the ACIA. It is the system's interface between the user programs and the ACIA (simulated) hardware.

Constructor & Destructor Documentation

◆ DriverACIA()

DriverACIA::DriverACIA ( )

Constructor. Driver initialization.

Constructor. Initialize the ACIA driver. In the ACIA Interrupt mode, initialize the reception index and semaphores and allow reception and emission interrupts. In the ACIA Busy Waiting mode, simply inittialize the ACIA working mode and create the semaphore.

◆ InterruptReceive()

void DriverACIA::InterruptReceive ( )

Reception interrupt handler. Used in the ACIA Interrupt mode only.

Reception interrupt handler. Used in the ACIA Interrupt mode only. Reveices a character through the ACIA. Releases the receive_sema semaphore and disables reception interrupts when the last character of the message is received (character '\0').

◆ InterruptSend()

void DriverACIA::InterruptSend ( )

Emission interrupt handler. Used in the ACIA Interrupt mode only.

Emission interrupt handler. Used in the ACIA Interrupt mode only. Detects when it's the end of the message (if so, releases the send_sema semaphore), else sends the next character according to index ind_send.
